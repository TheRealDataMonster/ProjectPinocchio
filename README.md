# ProjectPinocchio
Making a real boy... Or a Pet... Or anything that has a soul

## Goal
The goal of this project is to ideate and experiment on how different types of context and sophisticated logic around AI and its physical form (a robot), may be able to emulate unique life-like experiences. Applications of interest are artistic as well as entertainment focused. Some utility driven experiments may be done to determine what types of information need to be incorporated to the "soul" to create a life-like experiences that have real-world impact beyond "putting on a show". It's really about figuring out a way to bring faimiliarity to technology. 

## V0
Starring Reachy-Mini, an adorable mini robot carriable with out two arms and 2 devs - Dean and Kevin, we wanted to show that non-humans can be capable of fitting into a very human environment and be human conversation. They can emulate familiar feelings such as joy, sadness, anger, fear, and disgust. Heck it can even help read things for Dean, who has dyslexia. Robots and AI can very much feel human and familiar. 

We also experienced some frustrations trying to work with technology, once because of the cost - with ElevenLabs, which we couldn't use in our work because we couldn't get the free credit but had to pay $10 just fora quick demo. And again with Voxtral, which we took painstaking hours implementing but couldn't make the latency requirement to be good enough for the demo.

As a note, we've used Mistral Vibe to help structure how to go about adding new additions to types of reactions it can create. It also helped tremendously in structuring the project file paths as well as how to do some compel git operations as we were forkign 2 separate projects then combining them.


## Based On 
This project has its start forking 2 existing repos.
1) https://github.com/pollen-robotics/reachy_mini_conversation_app - by pollen-robotics team. This repo is meant to be forked then some significant changes are meant to be made starting with adding options for variety of models then how it handles the "personality" of the robot.

2) https://github.com/arc-megicxyz-bot/arc-soul - as an inspiration for how to structure variety of contextual information and additional context that it must create based on the information that is recorded - idenity of the "soul", its origins, how and what it remembers, how self-aware it is, etc... This documentation and the way that the AI application handles the series of docs will continue to be more and more important as it improves upon previous versions. We've added docuemtns such as Motivations and Trauma as markdown files to represent types of words that trigger certain types of emotional reactions, which is an improvement from other versions - to create a more visceral feeling.
